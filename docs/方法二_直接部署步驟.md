# 方法二：直接部署到 Vercel（用手機開網頁）

照下面順序做即可，約 2～5 分鐘。

---

## 第一步：在 GitHub 建一個新倉庫

1. 打開 **https://github.com** 並登入。
2. 右上角 **+** → **New repository**。
3. **Repository name** 填：`solo-leveling-system`（或自訂）。
4. 選 **Public**，**不要**勾選 "Add a README file"。
5. 點 **Create repository**。
6. 建立後會看到一個網址，例如：  
   `https://github.com/你的帳號/solo-leveling-system.git`  
   **複製這個網址**，下一步會用到。

---

## 第二步：把本機專案推到 GitHub

在終端機執行（請把 `你的帳號` 換成你的 GitHub 使用者名稱）：

```bash
cd /Users/hokamyuentommy/Documents/solo-leveling-system
git remote add origin https://github.com/你的帳號/solo-leveling-system.git
git branch -M main
git add .
git commit -m "Deploy to Vercel"
git push -u origin main
```

- **若已經有 `origin`**（之前就連過 GitHub），改成只執行：
  ```bash
  cd /Users/hokamyuentommy/Documents/solo-leveling-system
  git add .
  git commit -m "Deploy to Vercel"
  git push -u origin main
  ```
- 若 push 時要求密碼，請用 **Personal Access Token**（GitHub → Settings → Developer settings → Personal access tokens 建立）。

---

## 第三步：在 Vercel 部署

1. 打開瀏覽器，前往 **https://vercel.com**。
2. 點 **Log In** → **Continue with GitHub**，授權 Vercel。
3. 登入後點 **Add New…** → **Project**。
4. 在列表找到 **solo-leveling-system**，點右側 **Import**。
5. 若沒看到 repo，點 **Adjust GitHub App Permissions** 把權限打開後再選一次。
6. 設定頁面**不用改**，直接點 **Deploy**。
7. 等約 1～2 分鐘，出現 **Congratulations** 後點 **Visit**。

你會得到一個網址，例如：**https://solo-leveling-system-xxxx.vercel.app**

---

## 第四步：用手機開啟

1. 手機打開 **Safari** 或 **Chrome**。
2. 網址列貼上第三步的 Vercel 網址。
3. 前往 → 即可使用。

之後隨時用手機開這個網址都可以，不需同 Wi‑Fi。

---

## 之後要更新網站

在本機改完程式後執行：

```bash
git add .
git commit -m "更新內容"
git push
```

Vercel 會自動重新部署，約 1～2 分鐘後在手機重新整理同一個網址即可看到更新。
